neurep

Per-neuron representation priors for sample-efficient compositional learning

Thesis

A simple per-neuron representation prior helps a dense network discover hidden task decomposition — and that helps sample efficiency.

Each hidden neuron $i$ carries a small learned (or fixed) descriptor $\phi_i$ — a position, a code, a membership vector. A single regularizer prices every weight by the structural distance between the neurons it connects:

$$R(W,\Phi)\;=\;\lambda \sum_{i,j} d(\phi_i,\phi_j)\,\cdot\,\psi(W_{ij})$$

Cheap-to-connect neurons end up in the same functional module; expensive ones decouple. The whole program is to test whether one such term, bolted onto an ordinary dense MLP / Transformer, buys a real reduction in the data needed to learn compositional tasks — and to nail down why, or why not.

Central question

Does adding one per-neuron representation prior to a standard dense model lower the sample budget needed to learn compositional tasks, and is the gain mediated by recovered decomposition — not by extra capacity, extra regularization strength, or an architecture change in disguise?

Latest findings

Most recent first. live entries link to a full write-up; others are notes with the write-up in progress.

  • Why the axon–dendrite prior packs into 2ᵈ sign-orthants report 2026-06-13

    The learned geometry is the optimizer's fixed point, not the task's. For the axon–dendrite wiring prior, $\partial\,\text{MSE}/\partial(a,D)\equiv 0$: AdamW erases the radial-force magnitude, leaving constant $\pm\eta$ steps that quantize axon positions onto the $2^d$ sign-orthant diagonals. The count follows a balls-in-bins law; the $d{=}2$ match to four modules is a coincidence, not recovery.

  • The axon–dendrite wiring prior on flat_modular report 2026-06-11

    It governs capacity, but it does not decompose. The asymmetric multi-prototype prior (one axon + M dendrites per neuron) beats NoReg — but only the anti-collapse density term earns the MSE. Axons are not variables, there is no subspace carving, and the learned geometry receives no gradient from task performance.

  • Representation-induced modularity: when can a code make weights task-modular? notes 2026-06

    A decoupling / collapse / localizability theory of when a per-neuron representation can force a weight matrix to be task-modular, with ~400 runs and a Task-B encoding regularizer. Headline: learned penalty-only codes lose to plain ℓ1 — the code needs a gradient path from the task, or it is decorative.

  • Functional modularity: function is task-forced, geometry is method-bought notes 2026-06

    Across cells × tickets × seeds: the function a module computes is forced by the task (Sobol-confirmed), but its geometry is bought by the method. Modules come out rotated, not neuron-aligned — axis-mass ≪ 1, so neuron-level ARI ≈ 0 even when the decomposition is real in a rotated basis.

  • Width sweep: the over-parameterization caveat notes 2026-06

    A rigorous redo of the flat-forensics width study: the Separator prior wins only at extreme width (128); a properly tuned ℓ1 is a much stronger baseline than first reported, and the apparent "width-4 destruction" was a hyperparameter artifact, not a property of the method.

  • Separator collapse: pruning-vs-collapse is a race, and timing decides the damage notes 2026

    The $R\to 0$ separator pathology is a race between pruning and collapse: a larger penalty $c$ delays collapse, and it is the timing — not the collapse itself — that decides how much accuracy is lost. Spike initialization turns out to be irreplaceable.

Theory

A constructive thread: when is a task decomposition even representable by a dense network, and at what cost? The program encodes a computation graph exactly into standard architectures and tracks how its discrete complexity $(D, N, r)$ — number of discrete values, number of nodes, and max number of parents — transfers to network size.

  • Notation for the computation graph, and an exact encoding into an MLP — complexity $(D, N, r)$ transfers to MLP depth and width. pdf
  • Exact encoding into a Transformer — how $(D, N, r)$ maps to token-embedding dimension, QK dimension, and number of heads.
  • (stretch) Regularizers that encourage this encoding, with a proof that the computation-graph encoding — up to equivalent variations — is the minimizer over dense networks subject to matching the graph's exact input–output behavior.
